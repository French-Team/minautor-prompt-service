# Audit — Page « Prompts » (Générateur)

**Fichier :** `pages/prompts.vue`
**Route :** `/prompts`

## État actuel

| Élément | Statut | Détail |
|---------|--------|--------|
| Formulaire de génération | ✅ **Fonctionnel** | UI complète, sélection identité, template, variables |
| `getCurrentIdentity()` | ❌ **Simulé** | Crée une identité en dur si pas d'identité existante |
| `generateComprehensivePrompt()` | ✅ **Fonctionnel** | Pipeline réel avec règles, identité, contexte |
| `createVersion()` | ❌ **Mémoire RAM** | VersionHandler stocke dans une Map — perdu au restart |
| Résultat affiché | ✅ **Fonctionnel** | ID, version, contenu, règles appliquées visibles |
| `PromptCustomization` composant | ✅ **Fonctionnel** | Personnalisation des préférences |
| Récupération contexte | ✅ **Réel** | `contextAnalyzer.analyzeProjectContext()` via le singleton |
| SSR `?folder=` support | ✅ **Fonctionnel** | `setWorkFolder(folderParam)` en SSR |

## Problèmes

- **Identités non persistées** : le formulaire recrée une identité à chaque génération si aucune n'existe
- **Versions non persistées** : `VersionHandler` stocke en mémoire RAM uniquement
- **Template saisi en dur** : pas de sélection depuis la bibliothèque de templates
- **Résultat non sauvegardé localement** : le prompt généré disparaît au refresh

## Notes pour dev-plan

- Connecter la sélection d'identité à un vrai service d'identité
- Permettre de choisir un template depuis la bibliothèque plutôt que de le saisir
- Sauvegarder les prompts générés (localStorage ou backend)
- Persister les versions sur disque ou base de données
