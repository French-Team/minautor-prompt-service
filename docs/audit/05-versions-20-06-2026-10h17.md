# Audit — Page « Versions »

**Fichier :** `pages/versions.vue` → composant `<VersionHistory />`
**Route :** `/versions`

## État actuel

| Élément | Statut | Détail |
|---------|--------|--------|
| Affichage historique | ❌ **Simulé** | Dépend du composant `VersionHistory` |
| `VersionHandler` | ❌ **Mémoire RAM** | `private versionStore = new Map<string, PromptVersion[]>()` |
| `createVersion()` | ❌ **Mémoire RAM** | Stocke dans la Map — perdu au restart serveur |
| `getVersionHistory()` | ❌ **Mémoire RAM** | Lit depuis la Map |
| Métriques d'usage | ❌ **Mémoire RAM** | `metricsStore = new Map<string, VersionUsageMetrics>()` |
| Feedback utilisateur | ❌ **Mémoire RAM** | `feedbackStore = new Map<string, UserFeedback[]>()` |

## Problèmes

- **Aucune persistance** : toutes les versions, métriques et feedbacks sont en RAM
- **Aucune seed data** : historique probablement vide à chaque démarrage
- **Dépend du générateur de prompts** : les versions ne sont créées que via le bouton "Générer" sur `/prompts`

## Notes pour dev-plan

- Persister les versions dans un fichier JSON, SQLite ou base de données
- Envisager un stockage Git-like (diff entre versions)
- Ajouter la possibilité de restaurer une version précédente
