# Audit — Page « Templates »

**Fichier :** `pages/templates.vue` → composant `<TemplateLibrary />`
**Route :** `/templates`

## État actuel

| Élément | Statut | Détail |
|---------|--------|--------|
| Affichage liste templates | ❌ **Simulé** | Le composant `TemplateLibrary` utilise probablement des données en mémoire |
| `TemplateLibraryService` | ❌ **Mémoire RAM** | `private templates = new Map<string, PromptTemplate>()` |
| Création template | ❌ **Mémoire RAM** | `storeTemplate()` → stocke dans la Map, perdu au restart |
| Édition template | ❌ **Mémoire RAM** | `updateTemplate()` → modifie la Map |
| Suppression template | ❌ **Mémoire RAM** | `deleteTemplate()` → supprime de la Map |
| Recherche templates | ❌ **Mémoire RAM** | `searchTemplates()` → filtre la Map côté serveur |
| Statistiques d'usage | ❌ **Mémoire RAM** | `TemplateUsageMetrics` stocké dans une Map |

## Problèmes

- **Aucune persistance** : tous les templates sont perdus au redémarrage du serveur
- **Aucun seed data** : la bibliothèque part probablement vide à chaque démarrage
- **Pas de stockage fichier ni base de données**

## Notes pour dev-plan

- Besoin d'un storage backend (JSON fichier, SQLite, ou base de données)
- Ajouter des templates par défaut pré-chargés au démarrage
- Envisager un export/import de templates (format JSON/YAML)
